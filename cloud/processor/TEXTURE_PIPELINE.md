# Texture Pipeline: OpenMVS TextureMesh

## Overview

Room textures are generated by projecting captured keyframe images onto the ARKit LiDAR mesh using OpenMVS TextureMesh. The pipeline produces a UV-mapped textured OBJ with a single texture atlas.

## Pipeline Steps

```
iPhone Capture (ARKit)
  → mesh.ply (312K faces) + keyframes (60 walk + 120 pano JPEGs) + camera poses (JSON)
  → upload to GCS as scan.zip

Cloud Processing
  1. Strip PLY classification property (ARKit adds per-face classification bytes)
  2. Decimate mesh to ~5K faces (trimesh quadric decimation)
     - Reduces seam boundaries between adjacent faces assigned to different cameras
     - Key finding: 312K faces produces severe texture duplication; 5K eliminates it
  3. Convert ARKit poses to COLMAP format (prepare_openmvs_input.py)
     - Camera convention: multiply cam-from-world by diag(1,-1,-1) on the left
     - ARKit: X-right, Y-up, -Z forward → COLMAP: X-right, Y-down, +Z forward
     - Mesh stays in ARKit world coordinates (no flip)
     - Deduplicate panoramic keyframe entries (metadata lists each pano frame twice)
  4. InterfaceCOLMAP → scene.mvs
  5. TextureMesh with --cost-smoothness-ratio 10.0
     - MRF per-face view selection (each triangle gets one best camera)
     - High smoothness ratio forces larger uniform texture patches
     - Global/local seam leveling disabled (crashes on OpenMVS v2.3.0 ARM)
  6. Fix MTL transparency (OpenMVS writes Tr=1.0, should be Tr=0.0)
  7. Output: textured.obj + textured.mtl + textured_material_00_map_Kd.jpg
```

## Key Parameters

| Parameter | Value | Rationale |
|-----------|-------|-----------|
| Mesh decimation | 5,000 faces | Eliminates texture duplication from per-face view disagreement |
| Camera model | PINHOLE | ARKit pre-corrects lens distortion |
| Smoothness ratio | 10.0 | Forces adjacent faces to use same camera view |
| Resolution level | 1 | Half-resolution texturing (adequate for room viewing) |
| Seam leveling | Disabled | OpenMVS v2.3.0 crashes; enable when upgraded |

## Coordinate Conventions

```
ARKit camera space:     X-right, Y-up,   looks along -Z
COLMAP camera space:    X-right, Y-down, looks along +Z
Conversion:             cam_from_world_colmap = diag(1,-1,-1) @ cam_from_world_arkit

ARKit mesh:             Y-up world coordinates (unchanged in pipeline)
Camera intrinsics:      fx=1428.38, fy=1428.38, cx=959.0, cy=725.4 (iPhone 13 Pro)
Image resolution:       1920 × 1440
```

## Known Issues

1. **Seam leveling crashes** on OpenMVS v2.3.0 with `unordered_map::at` when faces have no assigned view. Affects both ARM and x86 builds. Workaround: disable seam leveling.
2. **ARKit mesh gaps** at thin geometry (door frame edges, curtain rods). Not fixable in processing — requires better scan coverage.
3. **Orange patches** = mesh faces where no camera has a good viewing angle. Common at ceiling corners and behind furniture.
4. **MTL transparency bug** — OpenMVS writes `Tr 1.000000` (fully transparent). Must be corrected to `Tr 0.000000` after each run.
